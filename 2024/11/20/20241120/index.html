  • 不知道为什么还是不行，最后部署到了自己的服务器里面

  • 学到的一些东西：

    • 创建一个新的分支
      • git checkout –orphan docs
      • orphan 的意思是新建的分支与原分支没有任何关系，所以需要把原分支的内容复制到新分支中
    • 清空旧的文件
      • git rm -rf .
      • rm -rf 是递归删除文件和文件夹
    • 撤销 git rm -rf . (还没有提交)
      • git reset HEAD .
      • reset HEAD . 会将暂存区中的所有文件取消暂存，但不会影响工作区中的文件。
      • git checkout .
      • checkout . 会将工作区中的所有文件恢复到最近一次提交的状态。
    • 撤销 git rm -rf . (已经提交了)
      • git log 查看提交记录
      • git reset –hard 回退到指定提交记录
      • 注意：此操作会丢弃当前未保存的修改。

  • 查看当前代理

    • git config –global –get http.proxy
    • git config –global –get https.proxy

  • 取消代理

    • git config –global –unset http.proxy
    • git config –global –unset https.proxy

  • 设置远程分支并推送

    • git push -u origin docs
    • -u 参数的作用是将远程分支设置为本地分支的 “upstream”，这样在以后使用 git pull 或 git push 时，就不需要再指定远程分支了。
    • origin 是远程仓库的默认名称，docs 是要推送的分支名称。
    • upstream 是一个指向远程分支的指针，用于跟踪远程分支的状态。在推送时，git 会将本地分支的内容推送到远程分支，并将远程分支设置为本地分支的 upstream。

  • 查看远程分支

    • git branch -r

  • 查看本地分支

    • git branch

  • 切换分支

    • git checkout

  • 删除分支

    • git branch -d
    • git branch -D 强制删除

  • 查看当前分支

    • git branch

  • 自己的阿里云服务器上传

    • 将本地资源上传服务器
    • scp -r “C:\Users\Adam\Desktop\dist*“ root@8.138.1.181:/usr/share/nginx/html/dist/
    • sudo rm -rf /usr/share/nginx/html/dist （服务器删除 dist 文件夹及其内容）
    • sudo 用于获取管理员权限（如果你没有足够权限的话）。
      rm 是删除命令。
      -r 表示递归删除，适用于删除目录及其中的文件。
      -f 强制删除，避免确认提示。

  • 想用远程仓库的内容覆盖我本地的内容

    • git fetch origin
    • fetch origin 会从远程仓库获取最新的内容，但不会自动合并到本地分支。
    • pull origin 会从远程仓库获取最新的内容，并自动合并到本地分支。
    • git reset –hard origin/master
    • reset –hard origin/master 会将本地分支的内容重置为远程分支的内容，并丢弃所有未提交的修改。
